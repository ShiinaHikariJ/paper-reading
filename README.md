# Paper Reading Tracker

用于记录论文阅读、研究方向梳理和每周进度。当前重点围绕 **大模型与小模型协同、PHM / 故障诊断、数据增强、时间序列生成** 展开，尤其关注大模型如何参与机械故障数据增强、虚拟样本生成和轴承故障诊断。

## 仓库结构

| 部分 | 入口 | 说明 |
| --- | --- | --- |
| 论文清单 | [papers/paperlist.md](papers/paperlist.md) | 统一维护已读和待读论文，包含论文链接、方向、来源、笔记入口和一句话小总结。 |
| 阅读笔记 | [notes/](notes/) | 存放每篇论文的详细阅读记录，重点整理论文问题、方法、实验、创新点、局限性和对当前研究的启发。 |
| 主题梳理 | [topics/](topics/) | 按研究主题沉淀横向总结，用来把单篇论文串成方法脉络和研究问题。 |
| 周期记录 | weekly / log | 记录每周阅读、思考和研究推进情况，适合沉淀阶段性进展。 |
| 模板 | [templates/](templates/) | 存放论文笔记和周报模板，保证后续记录格式统一。 |

## 论文阅读清单

[论文阅读清单](papers/paperlist.md) 是这个仓库的主索引。它分为总表和分类表两部分：总表方便快速查找每篇论文，分类表则按研究用途组织文献，例如综述基础、时序生成、扩散模型、大模型辅助数据增强和大模型故障诊断。

每条记录通常包含：

- 论文编号；
- 论文原文链接；
- 年份、方向和来源；
- 对应阅读笔记；
- 小总结，用一句话概括论文主要做了什么和创新价值。

## 阅读笔记

[notes](notes/) 目录保存具体论文笔记。笔记不是简单摘抄，而是尽量围绕研究使用场景整理：

- 论文要解决什么问题；
- 方法结构和关键模块；
- 实验设置和主要结果；
- 创新点与局限性；
- 与机械故障诊断、少样本学习、数据增强或大模型协同的关系。

目前重点笔记包括 TimeGAN、GT-GAN、Diffusion-TS、Diff-MTS、FIDE、CPS、FaultDiffusion、LLM-CoTAugSCLFD、KG-MTD、DiagLLM 和 BearLLM 等。

## 主题梳理

[topics](topics/) 目录用于做跨论文的主题归纳。相比单篇笔记，这里更关注“方法线索”和“研究空白”，例如：

- 时间序列数据增强方法如何演进；
- 生成式模型如何从 GAN 走向扩散模型；
- 频域保持、极值保持和硬约束如何服务机械故障信号生成；
- 大模型在数据增强、知识引导和故障诊断中分别适合承担什么角色。

## 周期记录

周期记录用于保存每周阅读和研究进展。它适合沉淀阶段性问题、近期对话整理、候选研究方向、下一步计划和待补充文献。

这部分更像研究日志，帮助从“读过很多论文”过渡到“形成一个清晰可推进的研究问题”。

## 当前研究主线

### 1. 数据增强与故障数据生成

这条线从时间序列数据增强综述出发，经过 TimeGAN、GT-GAN、Diffusion-TS、Diff-MTS、FIDE、CPS 和 FaultDiffusion，关注如何生成真实、多样、可控并符合机械故障特征的时间序列数据。

核心问题是：

> 在故障样本不足时，如何生成既像真实数据、又保留故障频率、周期冲击和工况语义的样本？

### 2. 大模型与机械故障诊断

这条线关注大模型如何处理振动信号、包络谱、故障知识和诊断任务。相关论文包括 One Fits All、LLM-based Bearing Fault Diagnosis、DiagLLM 和 BearLLM。

核心问题是：

> 大模型应当直接诊断，还是与小模型、信号处理工具和领域知识协同工作？

### 3. 大模型辅助数据增强

这条线目前最接近候选研究方向。它关注让大模型提供故障知识、增强策略或生成条件，再由扩散模型、传统信号处理方法和小模型完成数值生成、物理验证与诊断评价。

核心问题是：

> 如何把大模型的领域知识转化为可执行、可验证、能提升下游诊断性能的数据增强机制？
