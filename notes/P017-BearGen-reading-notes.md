# BearGen 阅读笔记

## 1. 论文信息

- **题目**：BearGen: LLM-guided Signal Generation Framework for Bearing Fault Diagnosis
- **作者**：Jaeyoung Lee, Hyuna Jeon, Uiin Kim, Misuk Kim
- **期刊**：Advanced Engineering Informatics
- **卷期**：71
- **文章号**：104400
- **年份**：2026
- **DOI**：10.1016/j.aei.2026.104400
- **研究方向**：大模型引导的轴承振动信号生成、描述条件扩散模型、类别不平衡、少样本故障诊断、本地大模型部署

---

## 2. 一句话总结

> BearGen先利用大型语言模型根据故障标签和工况生成信号文字描述，再将该知识蒸馏到可本地部署的小型LLM中，由本地LLM结合振动信号和健康参考生成描述，最后以自然语言描述作为条件，引导潜空间扩散模型生成轴承故障振动信号。

---

## 3. 研究背景

工业故障诊断高度依赖高质量振动数据，但真实工业场景中存在以下问题：

1. **故障数据稀缺**：设备大部分时间处于正常状态，故障样本远少于健康样本。
2. **类别严重不平衡**：正常数据占多数，轻微故障、严重故障和稀有故障样本较少。
3. **数据采集代价高**：主动制造故障可能损坏设备，真实故障数据还需要专家标注。
4. **工业数据具有敏感性**：振动信号可能暴露设备参数、工况和生产状态，企业不愿将原始数据上传到云端大模型。
5. **传统生成模型控制能力有限**：
   - VAE可能丢失细粒度结构；
   - GAN容易训练不稳定、模式崩溃；
   - 普通扩散模型缺乏类别控制；
   - 仅用故障标签作为条件，提供的信息较粗糙。

作者认为，自然语言描述可以比单一类别标签提供更丰富的条件信息，例如同时表达：

- 故障位置；
- 故障严重程度；
- 工况；
- 周期冲击；
- 信号不规则程度；
- 振幅和频率变化。

---

## 4. 研究问题

论文主要回答三个问题：

### 4.1 如何利用LLM帮助生成轴承振动信号

LLM不直接输出连续振动数值，而是生成信号的语义描述，再将描述作为扩散模型条件。

### 4.2 如何在工业现场本地使用LLM

论文先利用云端大型LLM生成一批教师描述，再将知识蒸馏到Qwen2.5-1.5B，使后续描述生成可以在本地完成。

### 4.3 如何利用生成数据缓解少样本和类别不平衡

论文针对均衡、少样本和严重类别不平衡场景，生成额外故障数据，并测试其对下游1D-CNN分类性能的提升。

---

## 5. BearGen总体框架

BearGen包含三个阶段。

```text
阶段一：构建信号描述数据集

故障标签 + 运行工况 + 提示词
                ↓
            大型LLM
                ↓
          自然语言信号描述

阶段二：训练本地小型LLM

待描述振动信号 + 同工况健康参考信号
                ↓
            FCN编码器
                ↓
      对齐层映射为信号Token
                ↓
  信号Token + 标签 + 工况 + 提示词
                ↓
         本地Qwen2.5-1.5B
                ↓
          自然语言信号描述

阶段三：描述引导扩散生成

原始振动信号
      ↓
VAE编码至潜空间
      +
LLM生成的文字描述
      ↓
描述条件1D U-Net扩散模型
      ↓
生成潜变量
      ↓
VAE解码
      ↓
合成轴承振动信号
```

---

## 6. 问题形式化

原始数据集定义为：

\[
\mathcal D
=
\{(x_i,y_i,w_i)\}_{i=1}^{N}
\]

其中：

- \(x_i \in \mathbb R^L\)：振动信号；
- \(y_i\)：故障类别；
- \(w_i\)：运行工况；
- \(N\)：样本数量。

对每个样本生成文字描述：

\[
\tilde c_i
\]

再以该描述作为条件，生成新信号：

\[
\tilde x_i
\]

最终增强数据集为：

\[
\mathcal D_{\text{aug}}
=
\mathcal D
\cup
\{(\tilde x_i,y_i,w_i)\}_{i=1}^{K}
\]

## 7. 阶段一：信号描述数据集构建

### 7.1 数据选择

作者从原始数据中选择10,000个样本，并尽量保持原数据集类别分布。

\[
M=10000
\]

构造教师描述数据集：

\[
\mathcal D_{\text{desc}}
=
\{(x_i,y_i,w_i,Inst_i,c_i)\}_{i=1}^{M}
\]

其中：

- \(Inst_i\)：提示词；
- \(c_i\)：大型LLM生成的描述。

### 7.2 大型LLM输入

提示词由以下信息组成：

\[
p_i^{prompt}
=
[y_i,w_i,Inst_i]
\]

即：

- 故障标签；
- 工况；
- 描述指令。

作者使用20种不同句式，增强描述表达的多样性。

### 7.3 使用的教师模型

论文使用：

> gpt-4o-mini-2024-07-18

生成10,000条教师描述。

### 7.4 需要注意的问题

第一阶段的大型LLM并没有直接读取原始振动波形。

它获得的是：

- 已知故障标签；
- 已知工况；
- 提示词。

因此，这些描述更准确地说是：

> 根据故障类别和工况生成的典型信号知识描述。

而不是对某一条具体波形进行真实分析后得到的样本级描述。

---

## 8. 阶段二：本地LLM训练与描述生成

### 8.1 本地LLM骨干

本地模型以BearLLM架构为基础，语言模型使用：

> Qwen2.5-1.5B

通过LoRA进行微调。

主要训练参数：

| 参数 | 设置 |
|---|---:|
| LoRA rank | 4 |
| scaling factor | 32 |
| dropout | 0.1 |
| learning rate | \(1\times10^{-4}\) |
| batch size | 1 |
| gradient accumulation | 4 |
| epochs | 50 |

### 8.2 健康参考信号

本地模型不仅输入待描述信号 \(x_i\)，还输入同工况健康参考信号：

\[
x_i^{ref}
\]

参考信号要求与当前信号具有相同：

- 通道；
- 转速；
- 负载；

且标签为健康状态。

FCN同时处理两者：

\[
V_i
=
FCN(x_i,x_i^{ref})
\]

其目标是学习：

> 当前信号相对于正常状态发生了什么变化。

这有助于：

- 抑制工况差异；
- 突出故障特征；
- 提高跨数据集和跨工况表示的一致性。

### 8.3 信号Token对齐

FCN输出特征图：

\[
V_i
\in
\mathbb R^{d_{fcn}\times t_{fcn}}
\]

先展开：

\[
v_i^{flat}
=
Flatten(V_i)
\]

再经过两个线性层：

\[
h_i^{(1)}
=
ReLU(W_1v_i^{flat}+b_1)
\]

\[
p_i
=
Softmax(W_2h_i^{(1)}+b_2)
\]

其中 \(p_i\) 可理解为故障类别混合系数。

然后映射到LLM嵌入维度：

\[
e_i^{vec}
=
W_3p_i+b_3
\]

再重塑为信号Token序列：

\[
E_i
\in
\mathbb R^{n_{token}\times d_{LLM}}
\]

最终输入LLM的提示序列为：

\[
q_i^{prompt}
=
[y_i,w_i,Inst_i,E_i]
\]

### 8.4 语义初始化

为了缩小振动信号与文本嵌入之间的模态差距，投影矩阵 \(W_3\) 使用预定义故障类别文本嵌入初始化。

### 8.5 LoRA训练目标

本地LLM学习模仿教师模型生成的描述：

\[
\mathcal L_{\text{LoRA}}
=
\frac{1}{M}
\sum_{i=1}^{M}
\mathcal L_{CE}
\left(
f_{\text{local}}(q_i^{prompt}),
c_i
\right)
\]

训练完成后，为全部样本生成描述：

\[
\tilde c_i
=
f_{\text{local}}(q_i^{prompt})
\]

构建扩展数据集：

\[
\mathcal D'
=
\{(x_i,y_i,w_i,Inst_i,\tilde c_i)\}_{i=1}^{N}
\]

## 9. 阶段三：描述引导的潜空间扩散模型

### 9.1 为什么使用潜空间扩散

原始振动信号维度较高，直接在原始空间扩散计算成本较高。

论文先使用VAE将信号压缩到低维潜空间，然后在潜空间中执行扩散。

### 9.2 VAE结构

编码器：

\[
z_{i,0}
=
E_{\phi}(x_i)
\]

解码器：

\[
\tilde x_i
=
D_{\psi}(z_{i,0})
\]

VAE损失包括：

\[
\mathcal L_{\text{VAE}}
=
\|x-\tilde x\|_2^2
+
\lambda_{\text{KL}}D_{\text{KL}}
+
\lambda_{\text{STFT}}\mathcal L_{\text{STFT}}
\]

三部分分别为：

1. 时域重构损失；
2. KL散度；
3. STFT频域损失。

### 9.3 VAE训练参数

| 参数 | 设置 |
|---|---:|
| batch size | 128 |
| 最大epochs | 300 |
| early stopping patience | 20 |
| STFT window | 1024 |
| hop size | 256 |
| \(\lambda_{\text{STFT}}\) | 0.1 |
| \(\lambda_{\text{KL}}\) | 从0逐步升至 \(10^{-4}\) |

### 9.4 文本描述编码

本地LLM生成的描述通过：

> all-MiniLM-L6-v2

编码为384维向量：

\[
e_{\tilde c_i}
\in
\mathbb R^{384}
\]

### 9.5 扩散训练

潜变量逐步加入高斯噪声，U-Net学习预测噪声：

\[
\mathcal L_{\text{diff}}
=
\mathbb E
\left[
\|
\epsilon-
\epsilon_{\theta}(z_{i,t},t,e_{\tilde c_i})
\|_2^2
\right]
\]

扩散模型参数：

| 参数 | 设置 |
|---|---:|
| backbone | 1D U-Net |
| input channel | 1 |
| sequence length | 128 |
| base channels | 64 |
| channel ratios | (1, 2, 4, 8) |
| diffusion steps | 1000 |
| noise schedule | cosine |
| condition dropout | 0.3 |
| batch size | 16 |
| learning rate | \(1\times10^{-4}\) |
| epochs | 500 |
| early stopping patience | 30 |

### 9.6 Classifier-Free Guidance

推理时：

\[
\epsilon_{\text{final}}
=
\epsilon_{\theta}(z_t,t,\varnothing)
+
s
\left[
\epsilon_{\theta}(z_t,t,e_c)
-
\epsilon_{\theta}(z_t,t,\varnothing)
\right]
\]

其中 \(s\) 为文本指导强度。

最终通过VAE解码：

\[
\tilde x_i
=
D_{\psi}(\tilde z_i)
\]

## 11. 数据生成策略

### 11.1 均衡数据集

DIRG、JUST和NCEPU被视为均衡数据集，生成信号数量约为原始数据集规模的两倍。

### 11.2 不平衡数据集

CWRU、PU、HIT、IMS和XJTU属于不平衡数据集。

## 14. 主要实验结果

### 14.1 生成质量

BearGen在7个数据集上取得最低FID。

| 数据集 | CFG-DDPM FID | BearGen FID |
|---|---:|---:|
| CWRU | 50.48 | **40.13** |
| PU | 14.73 | **11.65** |
| IMS | 79.23 | **72.70** |
| XJTU | 78.02 | **65.21** |
| DIRG | 19.72 | **19.51** |
| JUST | 36.43 | **34.63** |
| NCEPU | 64.72 | **50.20** |

KID方面，BearGen除XJTU外均为最佳。

### 14.2 波形与频域表现

论文观察到：

- GAN容易压缩振幅、丢失局部冲击；
- DDPM分布覆盖更广，但振幅控制不稳定；
- CFG-DDPM可能高估方差；
- BearGen的时域范围、FFT和包络谱更接近真实信号。

## 16. 指导强度消融

论文设置：

\[
s\in\{0,0.5,1,1.5,2,2.5\}
\]

结果显示：

- \(s\)过小，描述作用不足；
- \(s\)过大，可能损害多样性；
- 多数数据集在 \(s=1\) 或 \(1.5\) 时较好；
- 最终统一采用 \(s=1\)。

## 20. 论文主要贡献

1. **将LLM真正接入轴承信号生成流程**  
   LLM描述直接成为扩散模型条件。

2. **提出描述引导扩散模型**  
   自然语言描述比单一标签包含更多语义。

3. **支持本地部署**  
   通过知识蒸馏和Qwen2.5-1.5B降低外部数据泄漏风险。

4. **使用健康参考信号**  
   强调健康状态到故障状态的变化。

5. **实验规模较大**  
   覆盖8个数据集、少样本、不平衡、频谱分析和计算成本。

## 21. 客观局限

### 21.1 教师LLM没有真正读取信号

教师LLM只接收标签、工况和提示词，因此描述更接近类别知识，而不是样本级信号分析。

### 21.2 描述有效性实验存在标签泄漏风险

本地LLM输入包含真实故障标签，再使用描述辅助预测同一标签，可能产生循环监督。

### 21.3 缺少结构化条件的严格对照

没有完整比较：

1. 类别标签；
2. 类别 + 工况向量；
3. 固定模板文本；
4. 人工专家描述；
5. GPT描述；
6. 本地LLM描述。

因此，不能完全确认提升来自LLM推理，而不是更多条件信息。

### 21.4 物理一致性验证不足

论文没有系统评价：

- BPFI；
- BPFO；
- BSF；
- FTF；
- 冲击周期；
- 谐波与边带；
- 严重程度与能量关系。

### 21.5 序列长度较短

扩散序列长度仅128，远短于常见1024、2048或4096点轴承信号片段。

### 21.6 并非全流程完全本地

教师数据构建阶段仍依赖GPT-4o-mini。

### 21.7 未充分验证跨数据集生成泛化

主要是在各数据集内部训练和生成，尚未证明未见工况、未见设备生成能力。

### 21.8 复现成本高

需要同时训练：

- 本地LLM；
- FCN与对齐层；
- VAE；
- 文本编码器；
- 扩散U-Net。

## 22. 与相关工作的区别

## 23. 对当前研究方向的启发

### 23.1 从自由文本转向物理结构化条件

可以进一步使用：

- 轴承几何参数；
- BPFI；
- BPFO；
- BSF；
- FTF；
- 冲击周期；
- 谐波；
- 边带；
- 共振频带；
- 严重程度。

### 23.2 无标签泄漏的描述生成

测试时不输入真实标签，只输入：

- 原始信号；
- 健康参考；
- 工况；
- FFT结果；
- 包络谱结果。

### 23.3 显式故障残差生成

\[
x_{\text{fault}}
=
x_{\text{healthy}}
+
\Delta x_{\text{fault}}
\]

让扩散模型只生成故障残差。

### 23.4 物理验证闭环

生成信号后，通过：

- BPFI/BPFO计算；
- 包络谱分析；
- 冲击周期检测；
- 分类器结果；

反馈调整条件和指导强度。

### 23.5 跨工况和跨数据集生成

进一步研究：

- 已见转速生成未见转速；
- 已见负载生成未见负载；
- CWRU训练、Paderborn测试；
- 人工故障迁移到真实故障。

## 24. 总体评价

### 核心价值

BearGen是目前“大模型 + 扩散模型 + 轴承数据增强”路线中非常直接和完整的代表工作。

### 最有说服力的地方

- 覆盖8个轴承数据集；
- 对比多个GAN和扩散模型；
- 分析FFT、包络谱和相位多样性；
- 严重不平衡实验中F1提升明显；
- 考虑工业数据安全与本地部署。

### 最值得警惕的问题

- 教师描述没有真正读取信号；
- 描述分类实验存在标签泄漏；
- 未证明LLM文本优于等信息量结构化条件；
- 没有严格机械物理约束；
- 序列长度仅128；
- 未充分验证未见工况和跨数据集生成；
- 计算成本较高。

## 25. 阅读结论

BearGen已经完成：

```text
LLM生成信号描述
      ↓
描述作为扩散条件
      ↓
生成轴承故障振动信号
```

因此，以下内容已经不适合单独作为主要创新：

- 首次利用LLM引导轴承信号生成；
- 将自然语言作为扩散模型条件；
- LLM与扩散模型结合进行轴承数据增强。

更适合继续研究：

1. 无标签泄漏的信号描述；
2. 物理结构化条件；
3. 健康参考驱动的故障残差生成；
4. BPFI/BPFO等物理约束；
5. 生成—验证—调整闭环；
6. 跨工况、跨数据集生成泛化。
