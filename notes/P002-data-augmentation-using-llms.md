《Data Augmentation using LLMs》全文总结
一、论文基本信息

论文题目为 Data Augmentation using LLMs: Data Perspectives, Learning Paradigms and Challenges，发表于 Findings of ACL 2024。

这是一篇关于“大语言模型用于数据增强”的综述。论文不只是归纳“LLM怎么生成数据”，而是从两个层面建立完整框架：

数据视角：LLM具体对数据做了什么；
学习视角：生成的数据如何继续训练大模型或小模型。

论文还总结了数据污染、可控生成、多模态、隐私保护等五类核心挑战。第3页图1给出了全文分类体系。

二、一句话概括

这篇论文的核心思想是：

大语言模型不仅是最终需要训练的模型，也可以充当数据生成器、标注器、改写器、评价器和“教师模型”，以较低成本构建大量训练数据，再用这些数据训练其他大模型或任务专用小模型。

传统数据增强一般是对已有数据做局部变化，例如图像旋转、文本同义词替换、时间序列加噪声；而LLM数据增强更接近：

理解任务要求→生成或修改数据→自动标注、解释和评价→训练下游模型

因此，LLM带来的不是某一种增强算法，而是数据生产方式的变化。

三、论文为什么研究LLM数据增强

高质量训练数据存在几个问题：

人工采集和标注成本高；
专业领域数据难以获得；
人工标注存在错误和主观差异；
随着模型规模增大，对数据量的需求不断增长；
一些低资源语言、医疗、法律等领域缺少足够数据。

传统文本增强通常依赖：

同义词替换；
回译；
随机删除；
模板生成；
基于规则的变换。

这些方法往往难以真正理解句子的语义，容易出现标签改变、语句不自然、增强样本高度重复等问题。

LLM具有较强的语言理解、少样本学习、指令遵循和内容生成能力，因此可以通过少量示例快速生成大量结构完整、语义较自然的合成数据。

论文认为，LLM使数据增强从“机械修改原数据”逐渐转向“根据任务需求创造新数据”。

四、论文的总体分类框架

全文将LLM数据增强分为两个主要视角。

1. 数据视角

按照LLM如何处理数据，分成四类：

类型	核心作用
Data Creation	从少量示例出发，创造新的数据
Data Labeling	为已有无标签数据生成标签
Data Reformation	改写、变换已有数据，得到新版本
Co-annotation	人类和LLM协同完成标注
2. 学习视角

按照生成数据如何参与训练，分成：

生成式学习
监督指令学习；
上下文学习；
对齐学习。
判别式学习
为分类任务生成伪数据、伪标签；
为回归或评价任务生成分数。

这两个视角并不是互相独立的。

例如，LLM先对无标签数据进行标注，属于数据视角中的“数据标注”；之后用这些伪标签训练一个分类模型，又属于学习视角中的“判别式学习”。

五、数据视角下的四种方法
1. 数据创造：Data Creation

数据创造是让LLM根据少量示例、标签说明或任务描述，直接生成新的完整训练样本。

基本过程可以表示为：

任务说明+少量示例+目标标签→LLM→新样本

例如给LLM一个意图分类样本：

文本：帮我查询银行卡余额
标签：余额查询

再要求它生成多个具有相同意图但表达方式不同的样本：

我的卡里还剩多少钱？
查一下账户余额。
我想知道银行卡还有多少资金。
主要应用

论文介绍的数据创造应用包括：

医疗对话和医疗摘要；
通用对话系统；
情感支持对话；
意图识别；
推荐系统；
信息检索；
逻辑推理；
多语言常识推理；
指令微调数据生成；
思维链数据生成。
教师—学生知识迁移

数据创造还可以用于把大模型能力迁移给小模型。

例如：

问题→GPT-4生成答案和推理过程→训练小模型

大模型不仅提供最终答案，还生成推理步骤、解释或理由。小模型学习这些数据后，可能获得部分推理能力。

优点与问题

优点是扩展速度快、人工成本低，适合低资源场景。

但生成数据可能：

与原样本重复；
不符合目标标签；
过于模板化；
含有事实错误；
无法覆盖真实数据的长尾情况。

所以一般还要加入自动过滤、质量评价和人工抽查。

例如，让LLM判断文本情感：

输出：负面
主要应用

论文介绍了：

社交媒体和政治文本标注；
情感分类；
目标导向对话标注；
语音情绪标注；
跨语言数据标注；
临床实体识别；
视觉问答伪标签；
多模态科学问答推理标注。

部分研究发现，GPT-4或ChatGPT在一些文本标注任务上可以达到或超过众包标注者。

但是论文并不认为LLM标注必然优于人工标注。LLM可能因为提示词、类别定义、数据分布和领域知识不足而产生系统性错误，错误伪标签甚至会降低模型泛化性能。

因此需要控制：

哪些样本交给LLM；
哪些样本交给人类；
LLM置信度是否可靠；
是否采用多个模型交叉验证；
是否使用少量人工标签校准。
3. 数据重构：Data Reformation

数据重构是在保持主要标签或任务语义的情况下，对已有样本进行改写和变换。

典型方式包括：

同义改写；
句式变化；
文本扩写或压缩；
生成反事实样本；
混合多个已有样本；
改变部分属性；
保留实体标签的释义生成；
根据文本描述修改图像。

例如：

原始：这家餐厅的服务非常差。
标签：负面

改写：服务人员的态度让我非常失望。
标签：负面

它与数据创造的差别在于：

数据创造可以从任务要求直接生成全新样本；
数据重构以某个已有样本为基础，生成它的变体。
反事实增强

论文特别介绍了反事实数据：

原始：这部电影十分精彩。
标签：正面

反事实：这部电影非常无聊。
标签：负面

反事实增强通过改变少量关键因素，同时改变对应标签，可以帮助模型学习真正决定预测结果的因素，而不是依赖表面相关性。

这种方法有助于提高：

分布外泛化；
鲁棒性；
公平性；
对偏差的抵抗能力。
4. 人机协同标注：Co-annotation

协同标注把人类和LLM结合起来，而不是让LLM完全替代人工。

典型策略是：

简单、低不确定样本→LLM标注
复杂、高不确定样本→人工标注

论文介绍的CoAnnotating方法会根据LLM的标注不确定性，把不同样本动态分配给人类或模型。

LLM还可以向人工标注者提供：

关键词；
判断依据；
标签解释；
可能存在的歧义；
初始建议标签。

这样可以提高标注速度、标注一致性和标注者信心。

论文认为，人机协同可能比“全人工”或“全LLM”更加实际，因为它能够在成本和质量之间取得平衡。

六、学习视角：LLM如何作为教师

论文第6页图2把LLM数据增强放进了教师—学生学习框架：

教师LLM→生成数据、标签、解释或分数→训练学生模型

教师可以是GPT-4等大模型，学生可以是：

更小的语言模型；
分类器；
检索模型；
奖励模型；
其他任务专用模型。

作者进一步将这种学习分成生成式学习和判别式学习。

七、生成式学习
1. 监督指令学习

监督指令学习的数据形式通常是：

(instruction,x,y)

其中：

instruction：任务指令；
x：可选的上下文或输入；
y：期望输出。

例如：

指令：判断下面设备是否发生故障。
输入：振动信号分析结果……
输出：可能存在外圈故障。

LLM可以批量生成不同任务、不同输入和不同输出组合，形成指令微调数据集。

Self-Instruct、Alpaca等工作都体现了这一思路：利用已有模型生成指令和回答，再用这些数据训练另一个模型或继续训练自身。

2. 上下文学习

上下文学习是指模型不更新参数，而是通过提示词中的示例理解任务。

例如：

示例1：……
示例2：……
现在回答：……

LLM可以为另一个模型自动生成：

Few-shot示例；
多轮对话上下文；
时间推理问答；
知识图谱相关示例；
长上下文训练数据。

这种增强的重点不是简单增加独立样本，而是生成包含丰富上下文关系的数据，培养模型利用示例和上下文解决新问题的能力。

3. 对齐学习

对齐学习的目标是让模型输出更符合人类偏好、安全要求或任务目标。

LLM可以生成：

好答案和坏答案对；
偏好排序；
对比反馈；
安全和不安全回答；
人类偏好形式的合成数据。

例如生成：

因此，在对齐阶段，LLM不仅是生成内容，也可以模拟评价者，构造偏好训练数据。

这与知识蒸馏相似：大模型把隐含知识转化成合成数据，学生模型再从数据中学习。

2. 为回归和评价生成分数

LLM不一定输出离散类别，也可以输出连续分数或偏好值，例如：

奖励分数；
摘要质量分数；
回答相关性；

可表示为：

这些分数可以进一步用于：

强化学习奖励；
奖励模型训练；
生成结果筛选；
回归模型训练；
模型自动评价。

不过，LLM评价也可能存在位置偏差、长度偏好、自我偏好和提示词敏感性，因此不能默认LLM评分就是真实评价标准。

九、论文提出的五个主要挑战
1. 数据污染

LLM训练数据可能已经包含测试集或基准答案。

如果再用LLM生成与测试数据相似的增强样本，可能造成：

输入污染；
输入和标签同时污染；
模型记忆测试答案；
测试成绩虚高；
无法判断模型是否真正泛化。

未来需要研究合成数据污染的检测方法，并建立更动态、更新及时的测试集。

2. 可控数据增强

理想的数据增强需要精确控制某个属性，同时保持其他属性不变。

例如，只改变情感：

正面 → 负面

但应该尽量保持：

主题不变；
人物不变；
语法质量不下降；
非目标属性不被破坏。

当前LLM很难严格实现“只改变一个因素”。

此外，长期使用模型生成的数据继续训练模型，可能出现模型坍塌：

模型生成数据→新模型学习生成数据→分布越来越窄

最终模型会逐渐丢失真实数据中的长尾信息和多样性。

3. 文化感知与多语言增强

传统多语言增强经常只考虑词语和语法转换，却忽略：

地区习惯；
文化背景；
社会规范；
方言和惯用语；
不同文化下相同表达的不同含义。

未来的LLM增强需要生成既语言正确、又符合当地文化的样本，避免把某一种文化标准简单迁移到所有语言环境。

4. 多模态数据增强

LLM数据增强正在从文本扩展到：

图像；
音频；
视频；
图结构；
多模态组合。

难点不只是分别生成不同模态，而是保持模态间语义一致。

例如，一段设备故障描述、振动信号、时频图和故障标签应当互相对应。如果LLM修改了文字描述，却没有同步修改信号或图像，就可能产生错误数据对。

因此，多模态增强需要保证：

文本语义↔图像内容↔音频或信号结构↔标签
5. 隐私保护

医疗、金融和用户数据中可能包含敏感信息。LLM在生成数据时可能：

记忆并复现真实数据；
泄露姓名、地址、病历等信息；
从匿名信息中恢复个体身份；
生成与某个真实样本高度相似的数据。

未来可以结合：

数据匿名化；
差分隐私；
联邦学习；
数据审计；
输出泄露检测；
透明的数据使用政策。

但隐私保护越强，通常越可能降低数据质量，因此仍需要权衡。

十、论文的主要结论

作者认为，LLM使数据增强发生了三方面变化。

第一，数据增强从简单规则变换转向语义理解驱动的内容生成。

第二，LLM的角色从单一生成器扩展为：

数据创造者；
数据标注者；
数据改写者；
解释者；
评价者；
教师模型；
人工标注助手。

第三，LLM增强不再局限于传统分类任务，而是进入：

指令微调；
上下文学习；
模型对齐；
推理能力蒸馏；
奖励设计；
多模态学习。

但论文强调，合成数据并不是越多越好。真正决定效果的是：

数据质量+数据多样性+标签正确性+任务相关性+可靠筛选机制
十一、论文自身的局限

作者在结尾指出，LLM数据增强仍有几项明显限制：

生成数据质量依赖LLM原有训练数据和模型能力；
可能继承或放大偏见；
在专业、冷门领域容易出现知识错误；
大规模生成数据的计算和调用成本较高；
生成内容可能重复、空泛，无法真正增加数据多样性；
难以保证语义一致性和标签完全正确。

所以，LLM生成数据通常不能未经处理直接加入训练集，还需要：

生成→过滤→验证→人工抽查→下游训练
十二、对你轴承故障数据增强方向的意义

这篇论文主要讨论的是NLP和LLM训练数据，并不是直接研究轴承振动时间序列生成。因此，不能直接得出“用ChatGPT生成轴承振动信号”这一结论。

但它为你的“大小模型协同＋数据增强”提供了非常清晰的框架。

可以迁移的部分
1. 大模型作为数据标注器

输入传统模型提取的信号特征、频谱指标、工况信息和专家规则，让大模型辅助生成：

故障标签；
故障解释；
维修建议；
诊断报告。
2. 大模型作为数据生成控制器

大模型不直接生成振动波形，而是根据知识和任务要求生成增强条件：

故障类型：外圈故障
转速：1500 rpm
负载：2 kW
噪声强度：低
故障程度：轻微

再由时间序列生成模型、扩散模型或GAN按照这些条件生成振动信号。

可以理解为：

LLM负责语义和条件+小模型负责数值信号生成
3. 大模型作为筛选和解释模块

传统生成模型产生振动信号后，可以把提取出的频域和时频特征交给大模型，结合轴承机理知识判断：

是否符合目标故障；
故障特征频率是否合理；
信号是否明显违背物理规律；
该样本应该保留还是删除。

不过，大模型不能只靠自然语言凭空验证波形，需要先通过信号处理工具计算明确的数值特征。

十三、与上一篇时间序列综述的关系

上一篇论文回答的是：

时间序列数据可以怎么增强？

包括时域、频域、时频域、分解、GAN和自动增强。

这一篇论文回答的是：

大语言模型在数据增强流程中可以扮演什么角色，生成的数据又如何用于训练？

把两篇论文结合起来，可以得到一条与你课题更相关的研究路线：

LLM理解故障语义和工况要求→生成增强条件或增强策略→时间序列生成模型产生振动信号→信号处理工具验证物理特征→传统诊断模型训练

这比“让大模型直接生成一串轴承振动数值”更合理，也更符合老师所说的大小模型协同。

整篇论文最值得你记住的概括是：

LLM数据增强的价值，不只是生成更多数据，而是让大模型参与数据创建、标注、改写、筛选和评价，再把这些数据或知识迁移给下游模型。



是的，对轴承故障诊断而言，“数据重构”和“数据创造”确实最接近真正的数据增强应用。但两者的落地难度不同：

近期更实用：数据重构。
研究创新空间更大但风险更高：数据创造。

这篇综述本身主要研究文本数据，迁移到轴承振动信号时，不能直接照搬“让LLM改写文本”的方法，而应让大模型负责理解故障语义、制定增强条件和控制增强过程，由时间序列模型或信号处理算法真正操作数值信号。

一、数据重构：对轴承信号最容易落地

数据重构的基本思想是：

已有真实信号→受控变换→新的信号样本

例如已有一段外圈故障振动信号，可以进行：

加入符合现场环境的噪声；
改变信噪比；
时间平移或窗口裁剪；
幅值缩放；
局部时间伸缩；
频域幅值扰动；
时频图遮挡；
不同工况之间的风格迁移；
对故障冲击成分、背景成分分别增强。

它比完全从零生成信号可靠，因为增强样本始终来源于真实采集数据。

大模型在里面做什么

大模型不宜直接修改几千个振动采样点，而可以根据任务要求输出增强策略：

输入：
故障类型：外圈故障
转速：1500 rpm
负载：中等
目标：模拟强噪声工况，但保留外圈故障特征

大模型输出：
1. 保留外圈故障特征频率及其倍频；
2. 添加信噪比为5～10 dB的高斯噪声；
3. 幅值缩放范围为0.9～1.1；
4. 不进行信号翻转；
5. 增强后检查频谱峰值和包络谱。

然后由程序执行这些操作。

这就是：

LLM选择增强方法+信号算法执行增强
为什么它最适合你现在做

因为它有三个优势：

真实信号作为基础，物理可信性更高；
实验容易设计，可以与随机增强进行对比；
容易证明大模型发挥了什么作用，例如根据故障类别和工况动态选择增强策略。

所以，若你要先做一个能跑通、能验证的研究点，优先做大模型指导的受控数据重构更稳妥。

二、数据创造：创新性更强，但更难证明有效

数据创造是指从少量条件出发，生成全新的故障样本：

故障类别+工况+故障程度→生成模型→新振动信号

例如指定：

故障：内圈故障
转速：1800 rpm
负载：2 kW
故障程度：轻微
噪声水平：中等

然后生成一段对应的振动时间序列。

关键问题

大语言模型本身擅长生成文本，并不擅长直接生成高精度连续振动波形。让LLM直接输出：

0.012, -0.035, 0.082, ...

通常意义不大，因为它难以保证：

故障特征频率正确；
周期冲击存在；
幅值分布合理；
不同工况下信号变化符合规律；
生成信号不是随机数字；
标签与波形真正对应。

因此更合理的协同结构是：

LLM生成条件和知识约束→GAN/VAE/扩散模型生成振动信号→信号处理工具验证

也就是：

大模型负责“生成什么”；
时间序列生成模型负责“怎么生成波形”；
信号分析模块负责“生成得对不对”。
适合形成的研究问题

可以研究：

如何利用大模型生成的故障知识和工况条件，约束时间序列生成模型产生类别可控、工况可控且保持故障特征的振动信号？

这比简单使用GAN生成轴承数据更有“大—小模型协同”的特点。

但是验证必须包括：

频谱与包络谱检查；
故障特征频率保持情况；
真实与合成数据的分布距离；
多样性和重复性；
训练下游诊断模型后的性能；
跨工况泛化；
专家或规则验证。

否则容易被质疑为“合成信号提高了一点准确率，但不知道是不是有效故障信号”。

第一个点研究“真实信号如何受控重构”；
第二个点研究“新故障信号如何受知识约束生成”。

四、另外两类也不是完全没用
数据标注

当你有大量无标签或不完备标签的工业信号时，大模型可以结合传统模型结果、频谱指标和专家规则生成伪标签。

但它不能仅看原始数值就可靠判断故障，需要先由小模型和信号工具提取特征。

人机协同标注

这个对真实工业数据也有价值：

简单样本由模型自动标；
模型不确定的样本交给专家；
大模型生成判断依据和报告；
专家只负责确认或修改。

它比较适合系统应用或工程落地，但作为数据增强算法的核心创新，通常没有数据重构和数据创造直接。
