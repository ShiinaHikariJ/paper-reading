# Paderborn 900→1500 rpm 源域辅助生成实验阶段汇报

> 汇报目的：说明当前“利用900 rpm丰富数据辅助1500 rpm少样本生成”的实验进展、主要结果、发现的问题，以及下一步希望确认的研究方向。

---

## 1. 研究问题

当前实验围绕以下问题展开：

> 当900 rpm工况数据较多、1500 rpm工况数据较少时，能否先利用900 rpm数据预训练生成模型，再用少量1500 rpm数据进行适配，从而获得比“仅使用1500 rpm少样本直接训练生成模型”更好的故障信号生成与诊断效果？

实验中将两种方案进行对比：

```text
E2：仅使用1500 rpm少样本直接训练生成模型

E3：先使用900 rpm源域数据预训练，
    再使用相同的1500 rpm少样本进行目标域适配
```

最终统一在真实1500 rpm测试集上进行分类评估。

---

## 2. 数据与任务设置

数据集：

```text
Paderborn Bearing Data Center
```

使用三类故障：

```text
0 = Normal
1 = Outer race fault
2 = Inner race fault
```

信号处理：

```text
振动通道：vibration_1
原始采样率：约64 kHz
降采样：16 kHz
窗口长度：4096
```

数据规模：

```text
900 rpm源域训练集：
4500个窗口，1500个/类

1500 rpm目标训练池：
2700个窗口，900个/类

1500 rpm验证集：
900个窗口，300个/类

1500 rpm测试集：
900个窗口，300个/类
```

主要少样本设置：

```text
20-shot/类
每类20个真实1500 rpm训练样本
总计60个真实目标域样本
```

该协议属于：

```text
same-bearing cross-speed adaptation
```

即同一批轴承在900 rpm和1500 rpm下进行跨转速少样本适配，不涉及完全未知轴承泛化。

---

## 3. 第一阶段：简化版 Gen

早期实现采用：

```text
统计特征提取
→ Qwen生成技术描述
→ 文本Embedding
→ Autoencoder
→ 简化latent diffusion
→ 生成目标域信号
→ 下游分类
```

文本条件包含：

```text
故障类别
转速
RMS
Kurtosis
Crest Factor
高频能量比例
频谱质心
包络峭度
```

### 20-shot、五种子结果

```text
E2 Accuracy = 0.867778 ± 0.034418
E3 Accuracy = 0.873333 ± 0.029176
```

```text
E2 Macro-F1 = 0.866264 ± 0.037343
E3 Macro-F1 = 0.871571 ± 0.028492
```

平均差值：

```text
E3 − E2 Accuracy = +0.005556
E3 − E2 Macro-F1 ≈ +0.005307
```

即E3平均比E2高约：

```text
0.56个百分点
```

该阶段结果表现出弱正向趋势，但提升小于随机种子波动，不能认为源域辅助已经稳定有效。

## 4. 不同shot下的探索结果

seed42结果：

| Shot/类 | E2 Accuracy | E3 Accuracy | E3−E2 |
|---:|---:|---:|---:|
| 10 | 0.760000 | 0.726667 | -0.033333 |
| 20 | 0.883333 | 0.891111 | +0.007778 |
| 30 | 0.952222 | 0.923333 | -0.028889 |

对应Macro-F1：

| Shot/类 | E2 Macro-F1 | E3 Macro-F1 |
|---:|---:|---:|
| 10 | 0.763305 | 0.732625 |
| 20 | 0.882375 | 0.888326 |
| 30 | 0.952243 | 0.922666 |

阶段现象：

```text
10-shot：E3低于E2
20-shot：E3略高于E2
30-shot：E3低于E2
```

说明源域辅助效果并不稳定，并不是目标样本越少或越多时都必然有效。

可能原因是：

```text
10-shot时，目标数据不足以纠正900→1500 rpm偏差；

20-shot时，源域先验与目标适配之间相对平衡；

30-shot时，目标域直接训练已经较充分，源域偏差反而成为负担。
```

## 5. 第二阶段：高保真 Teacher-BearGen

为进一步接近BearGen的生成阶段，后续将方法改为：

```text
Teacher-BearGen without local LLM distillation
```

该版本保留：

```text
正常参考差异描述
Qwen教师描述
text-embedding-v4
压缩VAE
1D U-Net latent diffusion
Classifier-Free Guidance
```

未包含：

```text
本地LLM蒸馏
完整信号Token建模
完整三阶段部署流程
```

因此该方法不是完整BearGen复现，而是面向当前实验任务的高保真实现。

---

## 6. Teacher-BearGen 条件描述

每个样本的文本条件不再只包含简单类别词，而是包括：

```text
当前信号的统计特征
当前信号相对正常参考的变化
故障类别
转速
时域变化
频域变化
```

正常参考只使用训练数据构建，不使用验证集和测试集。

文本通过：

```text
qwen-plus
```

生成技术描述，再通过：

```text
text-embedding-v4
256维
```

得到扩散模型条件向量。

为了公平比较E2和E3，统一目标域阶段训练预算。

### E2

```text
300 epochs
lr = 1e-3

目标扩散：

lr = 2e-4
```

### E3

额外进行：

```text

lr = 1e-3

```

随后目标域适配仍为：

```text

300 epochs
lr = 1e-3

目标扩散：

```

两者统一使用：

```text

相同潜空间统计方式
相同生成数量
相同CFG

```

唯一额外差异是：

```text

```

## 10. VAE-only 消融

为了判断源域预训练到底作用在哪个组件，新增：

```text
E3-VAE-only
```

设置：

```text
900 rpm只预训练VAE
1500 rpm继续适配VAE
目标扩散模型随机初始化
不使用900 rpm扩散预训练
```

三种方法关系：

| 方法 | 900 rpm VAE预训练 | 900 rpm扩散预训练 |
|---|---:|---:|
| E2 | 否 | 否 |
| E3-VAE-only | 是 | 否 |
| E3-Full | 是 | 是 |

seed42结果：

```text
E2 Accuracy = 0.912222
E3-VAE-only Accuracy = 0.890000
E3-Full Accuracy = 0.905556
```

Macro-F1：

```text
E2 = 0.909642
E3-VAE-only = 0.884247
E3-Full = 0.901847
```

差值：

```text
VAE-only − E2 Accuracy = -0.022222
Full − E2 Accuracy = -0.006667
```

## 11. VAE-only 初步解释

seed42中：

```text
E3-VAE-only < E3-Full < E2
```

从VAE-only到Full：

```text
0.890000 → 0.905556
```

提高：

```text
+0.015556
```

这一现象初步提示：

1. 900 rpm VAE预训练可能将转速特定特征带入潜空间；
2. 这种潜空间偏移会影响1500 rpm目标生成；
3. 900 rpm扩散预训练并未进一步恶化结果，反而部分补偿了VAE迁移带来的损失；
4. 不同生成组件的跨转速可迁移性可能并不相同。

目前VAE-only只完成seed42，因此该结论仍需更多种子验证。

---

## 12. 当前可以得到的结论

### 结论一

```text
高保真Teacher-BearGen明显优于早期简化Gen
```

说明更细致的文本描述、VAE潜空间和U-Net扩散设计具有实际作用。

### 结论二

```text
完整900 rpm源域预训练尚未形成稳定优势
```

三种子结果分别为下降、持平和提升，平均差异接近0。

### 结论三

```text
源域迁移可能存在组件依赖
```

VAE-only结果提示VAE潜空间可能比扩散模型更容易保留转速特定偏差。

### 结论四

```text
源域数据量多，并不等于直接迁移一定有效
```

跨转速分布差异可能抵消源域预训练带来的生成先验。

---

## 13. 当前尚不能得到的结论

目前不能认为：

```text
老师提出的源域辅助思路已经被稳定验证
```

因为E3没有稳定超过E2。

也不能认为：

```text
源域辅助思路已经无效
```

因为：

```text
E3三种子均值与E2基本相同
seed44中E3高于E2
组件迁移可能存在正负效果抵消
```

同时不能把结果解释为：

```text
完整复现了BearGen原论文
```

因为当前实现不包含本地LLM蒸馏等完整流程。

---

## 14. 当前主要问题

### 1. 跨转速分布偏移

900 rpm和1500 rpm之间存在：

```text
故障特征频率移动
冲击间隔变化
频谱能量分布变化
潜空间分布偏移
```

直接迁移VAE和扩散模型可能带入源工况偏差。

### 2. 不同组件的迁移效果可能相反

当前结果可能表现为：

```text
VAE迁移产生负偏移
扩散预训练保留部分故障生成先验
二者最终相互抵消
```

### 3. 当前方法还没有形成稳定优势

三种子平均差异接近0，说明目前方法还需要进一步定位迁移机制，而不是继续简单增加训练轮数。

## 15. 下一步可选方向

### 方向一：继续做组件消融

补充：

```text
VAE-only：seed43、44
Diffusion-only：固定目标域VAE，只迁移源域扩散模型
```

形成完整矩阵：

| 方法 | 源域VAE | 源域Diffusion |
|---|---:|---:|
| E2 | 否 | 否 |
| VAE-only | 是 | 否 |
| Diffusion-only | 否 | 是 |
| Full | 是 | 是 |

该方向可以回答：

> 900 rpm源域信息究竟应该迁移到生成模型的哪一部分。

---

### 方向二：增加跨转速对齐

可考虑：

```text
阶次域转换
转速归一化
频率轴归一化
潜空间MMD对齐
潜空间CORAL对齐
只迁移低层卷积
重新初始化高层与条件层
```

其中优先考虑：

```text
阶次域 / 转速归一化
```

因为900 rpm和1500 rpm的主要差异首先体现在频率位置与冲击周期。

---

### 方向三：更换或增加第二个数据集

选择基线难度更适中、工况划分更明确的数据集，用相同方法验证：

```text
E2目标域直接生成
vs
E3源域辅助生成
```

该方向可以判断当前结果是否主要受Paderborn数据集和划分影响。

## 17. 阶段性总结

当前最客观的总结是：

> 在Paderborn 900→1500 rpm跨转速少样本任务中，早期简化Gen的源域辅助方案表现出弱正向趋势，但提升小于随机波动。高保真Teacher-BearGen显著改善了目标域直接生成和源域辅助生成的整体性能，但在公平目标训练预算下，E3与E2三种子平均性能基本相同，尚未观察到稳定的源域增益。进一步的VAE-only消融提示，900 rpm VAE预训练可能引入转速特定潜空间偏移，而源域扩散预训练可能保留部分可迁移的故障生成先验。下一步需要通过组件级消融和跨转速对齐，明确源域信息应如何迁移。
