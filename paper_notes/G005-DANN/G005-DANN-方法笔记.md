# Domain-Adversarial Training of Neural Networks 方法笔记

论文：Yaroslav Ganin et al., *Domain-Adversarial Training of Neural Networks*, JMLR 2016  
主题：无监督域适应，表示学习，反向梯度层，域不变特征  
阅读重点：方法动机、DANN 训练流程、GRL 设计、实验验证与复现要点

> 注：这篇论文严格来说是“域适应”而不是“域泛化”。它假设训练时可以看到无标签目标域数据；域泛化通常假设测试域训练时不可见。

## 0. 摘要原文翻译

作者提出了一种新的域适应表示学习方法，用于处理训练时数据和测试时数据来自相似但不同分布的情况。该方法直接受到域适应理论的启发：如果想实现有效的跨域迁移，模型的预测应当基于那些无法区分训练域（源域）和测试域（目标域）的特征。

该方法把这一思想实现到神经网络结构中：训练时使用有标签源域数据和无标签目标域数据，不需要任何目标域标签。随着训练进行，模型会促使特征同时满足两个性质：第一，对源域上的主要学习任务具有判别性；第二，对源域和目标域之间的分布差异不敏感。作者证明，只需要给几乎任意前馈模型增加少量标准层和一个新的梯度反转层，就能得到这种适应行为。增强后的结构可以用标准反向传播和随机梯度下降训练，因此能很容易地在深度学习框架中实现。

作者在两个不同的分类问题上验证了该方法：文档情感分析和图像分类，并在标准域适应基准上取得了当时的先进性能。此外，作者还在行人重识别应用中的描述子学习任务上验证了该方法。

## 1. 方法动机

### a) 作者为什么提出这个方法？

核心驱动力是：现实中有标签数据常常只在源域充足，而真正要部署的目标域往往没有标签，但两者分布又不一致。

典型例子包括：

- 用合成图像训练分类器，再部署到真实图像。
- 用电影评论训练情感分类器，再迁移到图书、电子产品或厨房用品评论。
- 在一个摄像头网络中训练行人重识别描述子，再迁移到另一个摄像头网络。

深度网络依赖大量标注数据，但目标域标注昂贵。作者希望利用“有标签源域 + 无标签目标域”，在一个统一的深度训练过程中学到能迁移的特征。

### b) 现有方法的痛点/不足

论文主要批评和补足了以下几类方法：

| 方法类型 | 痛点 |
|---|---|
| 固定特征上的浅层域适应 | 只在已有特征上做重加权、子空间映射或分布匹配，不能和深度特征学习联合优化。 |
| 自编码器类方法，如 mSDA | 学到的是鲁棒重构特征，分类器通常后训练；特征学习、分类器学习、域适应不是一个统一目标。 |
| 几何子空间对齐，如 SA | 依赖显式子空间映射，可能因为降维损失信息，对复杂视觉域偏移不够强。 |
| 有监督微调 | 需要目标域标签，而论文关注更难的无监督域适应。 |
| 显式分布距离匹配 | 常常需要设计特定距离或核空间，难以自然嵌入任意深度网络。 |

### c) 研究假设或直觉

如果一个特征既能预测源域标签，又让分类器分不出它来自源域还是目标域，那么基于这个特征训练出的任务分类器更可能迁移到目标域。

## 2. 方法设计

### a) 方法流程总结：输入 -> 处理 -> 输出

DANN 的输入包括：

- 源域样本：`S = {(x_i, y_i)}`，有类别标签。
- 目标域样本：`T = {x_j}`，无类别标签。
- 域标签：源域样本的域标签记为 0，目标域样本的域标签记为 1。

训练目标是输出一个在目标域上表现好的标签预测器。训练完成后，域分类器只用于训练，推理时一般丢弃，只保留“特征提取器 + 标签预测器”。

完整 pipeline 如下。

**第 1 步：把源域和目标域样本送入同一个特征提取器**

特征提取器记为 `G_f`。它可以是浅层全连接网络，也可以是 CNN、AlexNet 这类深度网络的一部分。

它把输入样本映射成中间表示：

```text
z = G_f(x)
```

这里 `z` 是后续两个分支共享的特征：一个分支预测任务标签，另一个分支预测样本来自哪个域。

**第 2 步：标签预测分支只用源域标签学习主任务**

标签预测器记为 `G_y`，输入特征 `z`，输出类别概率：

```text
y_hat = G_y(G_f(x_source))
```

损失是普通分类交叉熵，只在源域有标签样本上计算：

```text
L_y = classification_loss(y_hat, y_source)
```

这一步保证特征不能为了“域不变”而失去类别判别能力。也就是说，特征必须仍然能把源域类别分开。

**第 3 步：域分类分支同时看源域和目标域**

域分类器记为 `G_d`，输入同一个特征 `z`，输出样本来自目标域的概率：

```text
d_hat = G_d(G_f(x))
```

对源域和目标域样本都计算二分类损失：

```text
L_d = domain_classification_loss(d_hat, domain_label)
```

域分类器自己的目标是尽量分清源域和目标域。若它能轻松分清，说明当前特征仍然包含强域信息；若它分不清，说明源域和目标域在特征空间里更接近。

**第 4 步：在特征提取器和域分类器之间插入梯度反转层**

梯度反转层是 DANN 的关键。它前向传播时什么都不做：

```text
R(z) = z
```

所以域分类器看到的仍然是原始特征。

但反向传播时，它把来自域分类损失的梯度乘以负数：

```text
dR/dz = -lambda * I
```

这会产生一个非常重要的效果：

- 对域分类器参数来说：它仍然沿着降低 `L_d` 的方向更新，所以域分类器越来越会区分源域和目标域。
- 对特征提取器参数来说：来自域分类器的梯度被反向，所以特征提取器会沿着增大 `L_d` 的方向更新，让域分类器越来越难区分源域和目标域。

因此训练中形成了一个对抗过程：

```text
域分类器：努力找出域差异
特征提取器：努力隐藏域差异
标签预测器：努力保留类别信息
```

**第 5 步：联合优化三个模块**

整体目标可以理解为：

```text
最小化源域分类损失，同时最大化特征上的域分类损失
```

更直观地写：

```text
feature extractor: minimize L_y - lambda * L_d
label predictor:   minimize L_y
domain classifier: minimize L_d
```

论文从鞍点优化角度表述：对 `G_f` 和 `G_y` 做最小化，对 `G_d` 做最大化形式的等价目标。工程实现中，用 GRL 后就可以像普通网络一样反向传播，不需要手写复杂的 min-max 更新。

**第 6 步：训练时逐渐增强域对抗强度**

在深度实验中，作者没有一开始就强行要求域不变，而是让 `lambda` 从 0 逐渐增加到 1：

```text
lambda_p = 2 / (1 + exp(-gamma * p)) - 1
```

其中 `p` 是训练进度，从 0 到 1；`gamma = 10`。这样做的原因是：训练早期特征还很混乱，如果域对抗太强，可能会扰乱类别学习。先学基本类别特征，再逐步做域对齐，更稳定。

**第 7 步：推理时只用标签预测路径**

目标域测试样本经过：

```text
x_target -> G_f -> G_y -> predicted label
```

域分类器和 GRL 不再需要参与预测。

### b) 模型结构与模块协作

| 模块 | 输入 | 输出 | 作用 |
|---|---|---|---|
| 特征提取器 `G_f` | 源域或目标域样本 | 中间特征 | 学习既有类别判别性又尽量域不变的表示。 |
| 标签预测器 `G_y` | `G_f(x_source)` | 类别概率 | 用源域标签训练主任务分类器。 |
| 梯度反转层 GRL | `G_f(x)` | 前向输出不变 | 训练时反转域损失传给特征提取器的梯度。 |
| 域分类器 `G_d` | `GRL(G_f(x))` | 源域/目标域概率 | 判断样本来自哪个域，为特征提取器提供“隐藏域信息”的训练信号。 |

模块协作关系：

1. `G_y` 拉着 `G_f` 学有用类别特征。
2. `G_d` 试图从 `G_f` 的特征中识别域来源。
3. GRL 让 `G_d` 的成功反过来惩罚 `G_f`，迫使 `G_f` 抹掉域差异。
4. 最终 `G_f` 输出的特征既能支持源域分类，又让源域和目标域混在一起。

### c) 公式/算法的通俗解释

**H-divergence / Proxy A-distance**

论文借用了 Ben-David 等人的域适应理论：目标域误差可以被源域误差、源目标域差异以及一个共同可学习性项控制。直观上：

```text
目标域风险 <= 源域风险 + 源目标域分布差异 + 理想联合误差
```

所以如果源域分类误差低，并且源域/目标域在特征空间中难以区分，那么目标域误差也可能低。

Proxy A-distance 用一个二分类器来估计两个域有多容易被区分：

- 域分类器错误率接近 0：两个域很容易区分，差异大。
- 域分类器错误率接近 0.5：两个域难以区分，差异小。

DANN 直接把这个思想放进网络训练：让特征提取器生成能“骗过域分类器”的特征。

**目标函数**

普通源域训练只优化：

```text
min L_y
```

DANN 优化：

```text
min over feature/label parameters: L_y - lambda * L_d
min over domain-classifier parameters: L_d
```

`- lambda * L_d` 的含义是：对特征提取器来说，它希望域分类损失越大越好，也就是让域分类器越分不清越好。

**Algorithm 1 的含义**

浅层 DANN 的随机训练过程可以概括为：

1. 采一个源域样本，前向得到特征、类别预测和域预测。
2. 根据类别标签更新标签预测器和特征提取器。
3. 根据源域域标签更新域分类器。
4. 再采一个目标域样本，根据目标域域标签更新域分类器。
5. 对特征提取器，来自域分类器的梯度方向取反。
6. 重复直到收敛。

算法表面上有“最小化”和“最大化”的鞍点更新，但 GRL 让它在代码里变成一次标准反向传播。

## 3. 与其他方法对比

### a) 本方法和主流方法的本质不同

DANN 的本质不同在于：它不是在训练后对齐固定特征，也不是单独训练无监督表示，而是在端到端任务训练过程中，通过对抗式域分类信号直接塑造特征空间。

简洁地说：

```text
传统方法：先有特征，再做域对齐
DANN：一边学任务特征，一边让特征去域化
```

### b) 创新点与贡献度

| 创新点 | 贡献度 |
|---|---|
| 用域分类器近似源域/目标域可区分性 | 把域适应理论中的分布差异项变成可训练神经网络组件。 |
| 提出梯度反转层 GRL | 工程贡献很大，使对抗式域适应可以用普通反向传播实现。 |
| 统一特征学习、分类器学习和域适应 | 相比先训练表示再训练分类器的方法，训练目标更直接。 |
| 可插入几乎任意前馈网络 | 方法从浅层网络扩展到 CNN、AlexNet、Siamese 描述子网络。 |
| 多任务验证 | 不只做分类，也验证了行人重识别描述子学习。 |

### c) 更适用的场景

DANN 更适合：

- 有大量有标签源域数据。
- 有无标签目标域数据。
- 源域和目标域类别空间一致或高度重合。
- 域差异主要表现为外观、风格、背景、采集设备、评论领域等边缘分布差异。
- 目标域不太可能出现源域完全没有覆盖的类别或语义模式。

不太适合：

- 训练时完全看不到目标域数据的域泛化问题。
- 源域和目标域标签函数本身明显不同。
- 源目标域差异过大，导致“域对齐”可能把不同类别错误混合。
- 目标域无标签数据质量差或分布不代表真实测试环境。

### d) 方法对比表

| 方法 | 核心做法 | 优点 | 缺点 | DANN 的改进点 |
|---|---|---|---|---|
| Source-only | 只用源域标签训练 | 简单，不需要目标域数据 | 目标域分布变了就容易崩 | 用无标签目标域约束特征，减少域偏移。 |
| SVM/浅层分类器 | 在固定特征上训练分类器 | 稳定，适合小数据 | 不能自动学习跨域表示 | 端到端学习特征和分类器。 |
| mSDA | 用去噪自编码器学鲁棒表示 | 在文本域适应中强 | 目标是重构，不直接优化域不可分 | DANN 直接降低域可区分性，可与 mSDA 叠加。 |
| Subspace Alignment | 对齐源域/目标域子空间 | 实现简单，无需深度训练 | 依赖线性子空间，可能丢信息 | DANN 在网络内部非线性对齐。 |
| DDC/DAN 类分布匹配 | 用 MMD/均值嵌入等距离对齐 | 有明确分布匹配目标 | 距离形式需设计，可能不够自适应 | DANN 用可学习域分类器衡量差异。 |
| GAN 类对抗学习 | 生成器与判别器对抗 | 对抗思想强大 | 原始 GAN 目标是生成数据 | DANN 把对抗用于特征域不变，而非样本生成。 |

## 4. 实验表现与优势

### a) 作者如何验证有效性？

论文做了四类验证：

1. **二维 toy problem**：源域双月数据，目标域是旋转后的双月。可视化 DANN 如何调整决策边界和隐藏层表示。
2. **Amazon 评论情感分析**：四个领域（books、dvd、electronics、kitchen）之间做 12 个迁移任务。
3. **图像分类域适应**：MNIST -> MNIST-M、Synthetic Numbers -> SVHN、SVHN -> MNIST、Synthetic Signs -> GTSRB，以及 Office 数据集。
4. **行人重识别描述子学习**：在 VIPeR、PRID、CUHK/p1 之间迁移，验证方法不限于分类。

此外，作者还用 Proxy A-distance 和 t-SNE 可视化证明：DANN 学到的表示确实让源域和目标域更难区分。

### b) 代表性关键结果

**Amazon 评论数据集**

在原始 5000 维 unigram/bigram 特征上，DANN 相比 NN 和 SVM 整体更好；Poisson binomial test 显示 DANN 优于 NN 的概率为 0.87，优于 SVM 的概率为 0.83。

在 mSDA 表示上继续训练 DANN，DANN 优于 NN 的概率为 0.92，优于 SVM 的概率为 0.88，说明 DANN 可以和已有无监督表示学习互补。

部分准确率例子：

| 迁移任务 | DANN 原始特征 | NN 原始特征 | SVM 原始特征 | DANN+mSDA |
|---|---:|---:|---:|---:|
| books -> electronics | 0.733 | 0.747 | 0.748 | 0.804 |
| dvd -> electronics | 0.754 | 0.732 | 0.748 | 0.809 |
| electronics -> dvd | 0.738 | 0.733 | 0.726 | 0.781 |
| books -> kitchen | 0.779 | 0.778 | 0.769 | 0.843 |

注意：单个任务上 DANN 不总是第一，但跨任务统计更稳定。

**数字和交通标志图像分类**

| 源域 -> 目标域 | Source-only | SA | DANN | Train-on-target |
|---|---:|---:|---:|---:|
| MNIST -> MNIST-M | 0.5225 | 0.5690 | 0.7666 | 0.9596 |
| Syn Numbers -> SVHN | 0.8674 | 0.8644 | 0.9109 | 0.9220 |
| SVHN -> MNIST | 0.5490 | 0.5932 | 0.7385 | 0.9942 |
| Syn Signs -> GTSRB | 0.7900 | 0.8165 | 0.8865 | 0.9980 |

最突出的是 Syn Numbers -> SVHN：DANN 达到 0.9109，接近有标签目标域训练的 0.9220，覆盖了 source-only 到 train-on-target 差距的 79.7%。

**Office 数据集**

| 源域 -> 目标域 | Source-only | DANN | 代表性对比方法 |
|---|---:|---:|---|
| Amazon -> Webcam | 0.642 | 0.730 | DAN 为 0.685，DDC 为 0.618 |
| DSLR -> Webcam | 0.961 | 0.964 | DAN 为 0.960，DDC 为 0.950 |
| Webcam -> DSLR | 0.978 | 0.992 | DAN 为 0.990，DDC 为 0.985 |

优势最明显的是 Amazon -> Webcam，因为这是域偏移较大的场景。

### c) 哪些场景优势最明显？

优势最明显的场景是：

- 源域和目标域外观差异明显，但类别结构一致，例如 MNIST -> MNIST-M。
- 合成到真实迁移，例如 Syn Numbers -> SVHN、Syn Signs -> GTSRB。
- Office 中 Amazon -> Webcam 这种域差异较大的任务。
- 行人重识别中 PRID 与其他数据集差异较大时，适应带来的 CMC 曲线提升更明显。

证据包括：

- MNIST -> MNIST-M 从 0.5225 提升到 0.7666。
- Syn Signs -> GTSRB 从 0.7900 提升到 0.8865。
- Amazon -> Webcam 从 0.642 提升到 0.730，并超过当时对比方法。
- t-SNE 中适应后源域和目标域特征明显更混合。

### d) 局限性

论文中明示或隐含的局限包括：

1. **不是纯域泛化**：训练时需要无标签目标域数据。
2. **目标域差异过大时会失败**：论文提到 MNIST -> SVHN 无监督适应没有提升 source-only，说明对齐并非万能。
3. **可能发生负迁移**：如果不同域中类别条件分布不一致，强行对齐边缘特征分布可能把不同类别混在一起。
4. **域分类器结构需要经验选择**：论文承认域分类器结构较任意，调好可能进一步提升。
5. **lambda 调度很关键**：早期对抗太强可能破坏类别特征学习。
6. **目标域无标签样本必须代表测试分布**：如果目标域样本偏了，学到的域不变特征可能不服务真实测试。
7. **小数据下仍可能过拟合**：Office 实验中作者观察到目标域性能随训练继续会略微下降。

复现关键步骤：

1. 准备源域有标签数据和目标域无标签数据。
2. 为每个样本构造域标签：源域为 0，目标域为 1。
3. 搭建共享特征提取器 `G_f`。
4. 接标签预测头 `G_y`，只对源域计算任务损失。
5. 接域分类头 `G_d`，对源域和目标域都计算域分类损失。
6. 在 `G_f` 和 `G_d` 中间插入 GRL。
7. 使用混合 batch 训练：一半源域，一半目标域。
8. 训练时逐渐增大 `lambda`。
9. 测试时只用 `G_f + G_y`。

### b) 超参数、预处理、训练细节建议

**batch 组成**

论文深度实验使用 batch size 128，其中 64 个源域样本和 64 个目标域样本。这个设计很重要，因为域分类器需要同时看到两边样本。

**lambda 调度**

建议不要从一开始就设很大的对抗强度。可以使用论文公式：

```text
lambda_p = 2 / (1 + exp(-10 * p)) - 1
```

训练早期 `lambda` 接近 0，后期接近 1。

**学习率调度**

论文使用：

```text
mu_p = mu_0 / (1 + alpha * p)^beta
```

其中 `mu_0 = 0.01, alpha = 10, beta = 0.75`，并使用 momentum 0.9。

**域分类器容量**

域分类器不能太弱，否则无法提供有效对齐信号；也不能强到训练极不稳定。论文图像实验常用两层 1024 hidden units 的域分类器，MNIST 用较小的 100 hidden units。

**早停与无监督验证**

无监督域适应不能用目标域标签选模型。论文浅层实验使用 reverse validation：先源到目标训练，再用目标伪标签反向训练到源域，用源域验证集估计超参数好坏。

**实现 GRL**

在 PyTorch 中可用自定义 autograd function：

```python
class GradReverse(torch.autograd.Function):
    @staticmethod
    def forward(ctx, x, lambd):
        ctx.lambd = lambd
        return x.view_as(x)

    @staticmethod
    def backward(ctx, grad_output):
        return -ctx.lambd * grad_output, None
```

然后：

```python
features = feature_extractor(x)
class_logits = label_head(features)
domain_logits = domain_head(GradReverse.apply(features, lambd))
```

### c) 能否迁移到其他任务？

可以。DANN 的本质是“让中间表示对域不可辨”，所以只要任务可以写成“共享特征 + 任务头”，就可以迁移。

可迁移方向：

- **语义分割**：`G_y` 换成像素级分割头，`G_d` 可放在特征图级或输出图级。
- **目标检测**：对 backbone 特征或 proposal 特征加域分类器。
- **文本分类**：共享编码器输出句向量，域分类器判断文本来自哪个领域。
- **语音识别**：让声学特征对说话人、设备或场景域不可辨。
- **医学影像**：不同医院、扫描仪、协议之间做无标签目标域适应。
- **检索/重识别**：像论文第 5.3 节一样，把分类损失替换成度量学习损失。

迁移时要特别注意：域对齐位置应选择在既包含任务信息、又还保留域差异的特征层。太浅可能只对齐低级纹理，太深可能已经过度任务化。

## 6. 总结

### a) 一句话核心思想

反转域梯度，学习域不变特征。

### b) 速记版 pipeline

1. 同时读取有标签旧数据和无标签新数据。
2. 把两边数据变成同一种中间特征。
3. 用旧数据标签训练分类能力。
4. 训练另一个判断器分辨数据来自哪边。
5. 反向利用这个判断器，让中间特征隐藏来源差异。
